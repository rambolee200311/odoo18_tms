# 一期完成记录

## 覆盖场景（共8项）

| # | 场景 | 货物 | 流程 | 链路 |
|---|------|------|------|------|
| 1 | Terminal → 我方仓库 | 柜 / 托件 | 需求 → 计划 → 订单 | `transport.request` → Schedule → `pickup.plan` → `tlmp.transport.order` |
| 2 | Terminal → 客户地址 | 柜 / 托件 | 需求 → 询价 → 报价 → 订单 | `transport.request` → Inquiry → Quote → Order |
| 3 | 仓库 → 客户地址 | 托件 | 需求 → 询价 → 报价 → 订单 | 同上 |
| 4 | 客户A → 客户B | 托件 | 需求 → 询价 → 报价 → 订单 | 同上 |
| 5 | 我方仓库A → B（调拨） | 柜或托件 | 需求 → 计划 → 订单 | `transport.request` (warehouse_transfer) → Schedule → `pickup.plan` → Order |
| 6 | 客户 → 我方仓库（退货回仓） | 托件 | 需求 → 询价 → 报价 → 订单 | `transport_type = reverse_logistics`，同2 |
| 7 | 柜到仓后需换柜 | 柜级标记 | 运输订单容器明细 | `tlmp.transport.container`: `needs_swap` + `swap_location_id` |
| 8 | Depot / Terminal ↔ 仓库 空柜调拨 | 柜 | 柜务申请 → 计划 → 调柜订单 | `container.service.request` → `tlmp.transport.order` |

## 关键设计决策

- **流程严格二分**：计划驱动型（1、5、8） vs 商务报价型（2、3、4、6）
- **Schedule 仅用于港到仓和调拨**，其他场景不进日历
- **柜和托件分开建需求**，同一需求不混装
- **Terminal** 替代 Port 作为起点名称

## 模块结构

```
transport_logistics_management/
├── models/
│   ├── transport_request.py        # 运输请求（TMS 统一入口）
│   ├── transport_inquiry.py        # 承运商询价
│   ├── transport_quote.py          # 客户报价（+margin/cost）
│   ├── transport_order.py          # 运输订单（统一收敛出口）
│   ├── pickup_plan.py              # 提货需求（降级为排期子单据）
│   ├── schedule_plan.py            # 排期计划（+pickup_plan_id）
│   ├── transport_fee_type.py       # [已移除] 改用 charge.item
│   ├── transport_rate_base.py      # 费率底座（历史参考线）
│   ├── transport_fee_line.py       # 费用明细行（记录层）
│   ├── fee_base_inherit.py         # quote/order 挂载 fee.line
│   ├── container_service.py        # 空柜调拨申请
│   ├── transport_container.py      # 运输订单容器
│   └── ...
├── controllers/
│   └── pickup_schedule.py          # 日历排期 API
├── views/
│   ├── pickup_plan_views.xml       # 提货需求表单
│   ├── pickup_schedule_templates.xml # 排期页面
│   ├── container_service_views.xml  # 空柜调拨表单
│   └── transport_container_views.xml # 容器视图（含换柜字段）
└── static/
    └── src/
        ├── js/pickup_schedule.js   # 排期页面拖拽逻辑
        └── css/pickup_schedule.css # 排期页面样式
```

## 菜单

Transport → Operations:
- Transport Requests (5)           ← 全流程统一入口
- Pickup Requirements (10)
- Schedule Plans (15)
- Container Service (20)
- Inquiries (25)
- Quotes (30)
- Orders (40)
- Tracking (50)

Configuration:
- Fee Types      → world.depot.charge.item（全局费用主数据）
- Rate Bases     → transport.rate.base（历史费率参考）
- Fee Lines      → transport.fee.line（费用明细记录）
