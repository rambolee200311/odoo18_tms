# 荷兰跨境陆运外部承运商运输订单跟踪需求说明书
## 文档概述
### 1. 文档目标
搭建一套通用欧盟跨境/荷兰本地陆运订单全生命周期跟踪系统，适配码头入仓、仓间调拨、门到门派送、退货、空柜流转、柜换、保税跨境等全部物流场景；解决欧洲陆路运输多变临时变动、海关保税监管（MRN/T1）、ADR危险品合规、全链路审计追溯、时效管控、异常闭环、承运商对账等核心业务诉求；摒弃固定节点模式，采用动态运输事件记录，所有流程数据永久留存不可删除，支撑合规审计、时效分析、承运商绩效、AI延误预测。

### 2. 适用范围
荷兰本土及欧盟跨境公路陆运业务，对接第三方外部承运商；覆盖完税普通货物、保税转运货物、ADR危险品货物、空集装箱流转业务；面向调度、商务、财务、承运商对接、海关合规岗位。

### 3. 关联单据
CMR公路运单、POD签收凭证、MRN入境申报文件、T1保税转运单、ADR危险品申报单、空柜归还回执、承运商费用账单、海关查验记录、运输异常处理单

### 4. 核心设计思路
1. 废弃固定流程节点，采用**动态Transport Event运输事件**，支持换司机、换车、海关滞留、改期等各类临时变更；
2. 数据仅做状态标记，禁止物理删除，完整保留全链路轨迹满足欧盟物流审计；
3. 三层时间标准（计划/预估/实际）统一管控ETA/ATA，作为时效预警、承运商评分、AI预测底层数据；
4. 保税MRN/T1、ADR危险品为P0强制合规模块，适配荷兰海关监管要求；
5. 异常全生命周期闭环，未闭环异常不允许订单归档；
6. 区分计划驱动、报价驱动两类订单创建模式，分迭代落地，控制开发范围。

## 一、业务运输场景定义
### 1.1 全场景分类及创建规则
| 序号 | 运输场景 | 流程驱动类型 | 排期日历 | 询价报价流程 | 订单创建方式 | 特殊业务说明 |
|----|----|----|----|----|----|----|
| 1 | 码头Terminal → 我方仓库 | 计划驱动 | P1迭代 | 不启用 | 手动创建action_create | 保税/普通货均可，需空柜归还事件，支持MRN/T1 |
| 2 | 码头Terminal → 客户地址 | 商务报价驱动 | 不启用 | 启用 | 报价确认自动生成auto | 跨境保税派送，MRN/T1必填 |
| 3 | 我方仓库 → 客户地址 | 商务报价驱动 | 不启用 | 启用 | auto | 本地/跨境派送 |
| 4 | 客户A地址 → 客户B地址 | 商务报价驱动 | 不启用 | 启用 | auto | 跨境门到门保税转运 |
| 5 | 仓A ↔ 仓B仓间调拨 | 计划驱动 | P1迭代 | 不启用 | action_create | 内部库存流转，保税调拨需T1 |
| 6 | 客户地址 → 仓库退货 | 商务报价驱动 | 不启用 | 启用 | auto | 逆向跨境运输 |
| 7 | 柜到仓换柜业务 | 柜级扩展标记 | — | — | 依附现有订单扩展字段 | 仅更新新旧柜号，不单独生成订单 |
| 8 | 空柜堆场Depot ↔ 仓库 | 计划驱动 | 不启用 | 不启用 | action_create | 无货物，无需MRN/ADR |

### 1.2 两种订单驱动模式说明
1. **计划驱动订单**
仓库调度根据到货、库存、空柜排期手动新建，无客户询价报价链路；适用于码头入仓、仓间调拨、空柜流转，配套后续排期日历可视化功能。
2. **商务报价驱动订单**
商务对接客户发起询价，完成承运商比价、报价确认后系统自动生成正式运输订单；面向对外收费派送、退货、跨客户调拨业务。

## 三、动态运输事件 Transport Event 需求（P0核心）
### 3.1 事件类型枚举
| 事件类型 | 业务含义 | 适用场景 |
|----|----|----|
| PICKUP_ARRIVED | 车辆抵达提货地点 | 所有载货订单 |
| LOADING_COMPLETED | 货物装车完成 | 所有载货订单 |
| DEPARTED | 车辆驶离起始点 | 所有载货订单 |
| DELIVERY_ARRIVED | 车辆抵达收货地 | 所有载货订单 |
| DELIVERY_COMPLETED | 货物签收完成 | 所有载货订单 |
| EMPTY_RETURN | 空柜归还堆场 | 码头入仓、空柜流转场景 |
| CUSTOMS_CHECK | 海关查验 | 保税MRN/T1跨境订单 |
| OTHER | 其他临时变动事件 | 换司机、换拖车、堵车滞留、次日改派送等 |

### 3.2 单行事件固定字段
事件ID、关联订单ID、发生地点、事件类型、事件状态、跳过/取消原因、
planned_time计划时间、estimated_time预估ETA、actual_time实际ATA、
操作人（内部/承运商）、文字备注、现场附件。

### 3.3 事件状态管控规则（关键审计约束）
1. **禁止删除任何事件记录**，全程数据永久留存用于海关、企业审计；
2. 事件状态枚举：待执行 / 执行中 / 完成 / 跳过 / 取消；
3. 状态为「跳过/取消」时，`skip_cancel_reason` 为必填项，无理由无法保存；
4. 基础载货事件存在强制时序约束：抵达提货→装车→离开→抵达目的地→签收；
5. OTHER自定义事件无顺序限制，可任意新增，适配欧洲陆运各类突发变更；
6. 全部标准事件强制录入三层时间，OTHER事件预估、实际时间可选留空。

### 3.4 ETA/ATA三层时间业务价值
1. planned_time：仓库调度排期基准；
2. estimated_time：承运商实时反馈预估时效；
3. actual_time：真实落地时间，用于承运商时效评分、延误统计、AI延误预测、超时自动提醒底层数据源。

## 四、运输异常 transport_exception 闭环管理（P0）
### 4.1 异常全生命周期状态（强制闭环）
1. OPEN：异常刚登记，未启动处理；
2. PROCESSING：对接承运商、海关、客户处置中；
3. RESOLVED：问题处置完毕（换车、赔付、重新派送等）；
4. CLOSED：归档闭环；
业务强约束：订单标记「已闭环」前，系统强制校验所有异常状态必须为CLOSED，否则拦截归档操作。

### 4.2 异常基础字段
关联订单ID、绑定对应运输事件ID、异常发生时间、异常类型、
文字描述、承运商反馈、现场附件、处理责任人、处置方案、
理赔金额（如有）、resolved_time处置完成时间、closed_time闭环归档时间。

### 4.3 异常类型枚举
货物破损短缺、车辆故障、高速拥堵延误、错提货物、客户拒收、
柜身损坏、空柜逾期、海关扣货查验、T1保税转运逾期、MRN申报不符、
ADR司机无资质、危险品包装不合规。

## 五、途中额外费用 extra_charge 台账（P0）
### 5.1 业务逻辑
区分报价内基础运费与运输途中临时产生附加费，独立台账用于财务与承运商月度对账。
### 5.2 单行字段
关联订单ID、绑定对应运输事件ID、费用类型、金额、币种固定EUR、
计费单位、数量、费用产生原因、承运商收费凭证附件、费用承担方（客户/我方/承运商）。
### 5.3 费用枚举
滞柜费、滞车费、海关杂费、加急派送费、装卸附加费、空柜往返费、
货损处置费、ADR合规服务费、保税申报服务费。
### 5.4 结算规则
订单归档时自动汇总全部额外费用，同步关联至承运商月度对账单。

## 六、MRN保税申报 + T1转运单据管理（P0合规强制）
### 6.1 启用判定规则
自动展示字段：码头入仓、跨境派送、跨客户门到门、跨境退货、保税仓调拨；
隐藏字段：荷兰境内纯完税本地运输、空柜Depot往返。
### 6.2 MRN主申报单约束
一单对应一批跨境保税货物，多柜共用；记录海关起止地点、有效期，系统监控到期预警，超期自动生成保税逾期异常。
### 6.3 T1多行子表约束
一份MRN可关联多份T1转运文件，每份T1绑定对应柜号集合；记录海关转运截止时限，留存目的地海关签收凭证，核销时间为T1闭环标识。
### 6.4 模块联动能力
1. 新增CUSTOMS_CHECK海关查验事件，可绑定MRN/T1单号；
2. 保税类异常可直接关联对应T1单据；
3. CMR打印模板自动读取MRN、T1编号打印在单据保税备注栏；
4. 归档区单独分类存储MRN回执、T1纸质文件、海关核销证明。

## 七、ADR危险品字段管理（P0合规强制）
1. 订单勾选「含危险品」后，ADR子表全部字段必填，不可留空；
2. 同步校验司机ADR资质、专用运输车辆，缺失则禁止保存订单；
3. ADR信息自动同步至CMR随车单据，满足欧盟陆路运输随车查验要求；
4. 危险品相关违规纳入异常类型，发生后强制跟进处置闭环。

## 八、归档附件统一规范（P0）
1. CMR运单：一票一单号，支持套打生成PDF，扫描件自动归档；
2. POD签收单：DELIVERY_COMPLETED事件必填附件，无签收凭证无法标记签收完成；
3. 海关类：MRN申报回执、T1转运单、海关查验记录、核销文件独立分类；
4. 危险品：ADR申报单、危险品包装证明；
5. 通用：现场照片、司机证件、空柜回执、费用发票。

## 十、非功能性需求
1. **审计永久不可删除**：所有订单、事件、异常、费用、保税单据仅允许状态作废/关闭，物理数据永久留存，满足荷兰海关与企业内控审计；
2. **轻量化上下文适配**：运输事件、T1/ADR明细采用摘要快照模式，适配context_loader懒加载机制，控制Token开销；
3. 配置化枚举：运输场景、事件类型、异常类型、费用类型后台可配置，无需修改代码；
4. 系统联动：与CMR套打模块打通，MRN/T1/ADR数据自动填充打印模板；
5. 权限分层管控：
   - 调度：完整编辑订单、事件、异常、费用、保税单据；
   - 承运商账号：仅上传事件、附件、提交异常，不可修改费用与核心订单信息；
   - 财务：仅查看费用台账、报表，无编辑权限；
6. 数据追溯：所有新增、修改操作留存操作人、操作时间日志，可全链路回溯变更记录。

## 十一、核心业务约束汇总
1. 不允许删除运输事件，仅通过跳过/取消状态记录无关节点并留存原因；
2. 订单签收归档前，全部异常必须处于CLOSED闭环状态；
3. 跨境保税货物必须完整录入MRN+T1信息，否则无法完成签收闭环；
4. 含ADR危险品订单，司机资质、车辆、ADR申报字段强制校验；
5. T1转运超截止时限自动生成逾期异常，推送调度跟进核销；
6. DELIVERY_COMPLETED签收事件必须上传POD附件，否则状态无法改为完成；
7. 基础运输事件存在固定先后时序，系统拦截逆序操作。
