# CMR运单制作与打印需求说明书
## 1 整体方案架构
### 1.1 物理纸质载体
采用**预制印刷制式CMR空白单据纸**
- 纸张物理规格：`210mm × 310mm`，印刷版本：Version 8.0
- 纸面永久预印刷固定内容（系统无需绘制、渲染、绘制边框线条）
  1. 多语言表头：荷/英/法/德/西/意多语CMR运输单据标题、AVC标识
  2. 全域区块编号1~24、所有静态字段名称、红色框线、货物明细表格边框
  3. 页脚印刷厂商文字、各类备注区、收发/承运方签字栏、ADR证件粘贴空白区域
  4. 单据底部法定说明、签章说明固定印刷文本

### 1.2 系统输出方案
系统仅生成**纯文本数据PDF（无边框、无线条、无底色、无任何样式元素）**
依靠系统预设XY坐标点位，将业务文字精准叠加打印在预制CMR纸质对应格子内，标准套打实现。
对应业务文件关系：纸质实拍件=印刷固定底版；系统导出PDF=套打可变数据源。

## 2 系统PDF填充字段清单 & 分区打印规则
### 2.1 顶部单号区域
| 打印字段 | 数据来源 | 排版约束 |
|--------|--------|--------|
| NL开头8位CMR流水编号 | 承运商分配的CMR唯一编号 | CMR标识框内靠右对齐打印 |
| Order No（PINL前缀单号） | Odoo TMS运输订单主键编号 | 固定坐标写入18号区块 |
| REF（单号） | 荷兰仓库内部分拣入库参考号 | Order No下方同行区域打印 |

### 2.2 收发方信息区域（区块1 / 区块2 / 区块4）
1. 区块1（收货方）：固定为HOYMILES + 完整仓库物理地址、邮编、城市
2. 区块2（发货方）：公司全称、街道地址、邮编、城市、国家；兼容「西班牙境外跨境发货」「荷兰境内本地发货」两种场景
3. 区块4（过境国家）：国家名称
4. 排版规则：文本左对齐，地址支持自动换行适配印刷格子，统一使用固定等宽打印字体

### 2.3 实际承运人信息（区块16 / 区块17 / 区块23）
1. 打印内容：承运公司全称、CUI税号、详细地址、所属国家、车辆牌照号、车队备注信息
2. 区块23为承运人手写签字区：系统仅打印文字信息，签字位置留白，由现场手写签署

### 2.4 货物明细表格区域（印刷固定表格，系统填充单元格数据）
表格固定列：`Commodity`、`SKU`、`Qty(托盘数量)`、`GW/PCS(单件毛重kg)`、`GW(kg)本行总重`、`Points计费点数`
1. 支持动态明细行数：适配1行SKU、2行SKU自动填入印刷单元格
2. 表格底部汇总行：总托盘数Total Pallet、合计总毛重、合计Points由系统自动计算填充
3. Points字段允许数值为0（ADR场景 points=3*GW）

### 2.5 签收手写区域（区块22）
签收人姓名、签收日期全部线下手写录入，系统对此区域永久留白，不打印任何内容。

### 2.6 可选附加打印字段
仓库Dock码头编号为可选字段，打印在单据顶部空白区域。

## 3 PDF文件生成硬性规则
1. PDF画布尺寸严格匹配实体纸张：`210mm * 310mm`
2. 所有打印文本固化配置：字体、字号、X横向偏移值、Y纵向偏移值、对齐方式
3. PDF内部只生成文本对象，禁止生成绘图、线条、填充色等冗余元素
4. 单套坐标配置文件，统一兼容跨境运输、荷兰本地内陆运输两类业务单据
5. 全部业务数据取自Odoo TMS运输单、出库单、货品明细，支持界面手动微调修正数据

## 4 硬件打印执行需求
1. 打印模式：黑白激光打印，仅输出PDF内文本数据
2. 进纸方式：预制CMR空白纸置入打印机纸盒，PDF直接下发打印机完成套印，无需人工二次对位校准
3. 打印精度：坐标整体误差 ≤ 0.5mm，文字完整落在印刷格子内部，不得压边框、出格子
4. 一页一单：1份运输订单对应1份PDF、1张CMR单据；多SKU、多托盘货物统一打印在单页CMR内

## 5 业务流转配套需求
### 5.1 CMR生成触发时机
货物完成出库校验、装车确认操作后，在TMS系统内一键触发CMR数据PDF生成。

### 5.2 数据强校验规则
1. 总托盘数量 = 所有明细行Qty累加值，数值不一致时拦截CMR生成并弹窗报错
2. 单行总毛重 = 单件毛重 × 本行托盘数量；底部总毛重由各行数值自动求和校验
3. Order编号、REF编号设置非空强制校验，为空禁止生成单据

### 5.3 档案归档与补发
1. 生成的CMR PDF自动归档，绑定对应TMS订单档案形成关联关系
2. 支持PDF重复下载、重复打印，适配单据补打、回单补发场景
3. CMR编号全局唯一，系统管控不可重复生成相同编号单据

### 5.4 业务场景兼容
同一套模板兼容两类运输业务：
1. 欧盟跨境公路运输
2. 荷兰境内内陆短途陆运

## 6 非功能 & 架构适配约束
1. 套打XY坐标提供独立可视化配置界面，运维人员可直接微调偏移量，无需修改底层代码
2. PDF文件体量轻量化，配合现有`context_loader`快照+文件懒加载架构，不会产生额外无效Token消耗
3. 货物明细行数自适应，无需人工改动模板格式
4. 所有手写签章、批注区域系统固定留白，程序不会输出任何文字覆盖手写区域
