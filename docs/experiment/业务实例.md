## 实例 1：Terminal → 我方仓库（计划驱动）

### 实例 1A：鹿特丹码头 → 鹿特丹仓库（集装箱）

**背景**：
- 客户：ABC Trading B.V.
- 提单号：NKGA84065
- 到港码头：Rotterdam Maasvlakte Terminal
- 目的地：我方 Rotterdam 仓库
- 货物：2×40HQ 集装箱，电子产品

**系统操作流程**：

```
1. Transport Requests → 新建
   ├── Request Type: Plan-Driven
   ├── Cargo Type: Container
   ├── Destination: Terminal / Depot to Our Warehouse
   ├── Destination Warehouse: Rotterdam
   ├── Origin Terminal: Rotterdam Maasvlakte Terminal
   └── 保存

2. 在表单头点击 "Go to Schedule" → 进入排期日历
   ├── 系统自动将该 request 的货物加入左侧待排列表
   ├── 筛选 Warehouse → Rotterdam
   ├── 拖拽到 7月10日
   └── 排期后自动创建 pickup.plan + schedule.plan.schedule 记录

3. 返回 Transport Request → Scheduling tab
   ├── 可看到已创建的 Schedule Plans 和 Pickup Plans 子列表
   └── 打开 pickup.plan → 点 "Create Transport Order"
       └── 自动创建 Transport Order + fee.line (carrier_cost)
```

## 实例 2：Terminal → 客户地址（商务报价）

### 实例 2A：鹿特丹码头 → 阿姆斯特丹客户（集装箱）

**背景**：
- 客户委托从 Rotterdam 码头提一个 40HQ 柜送到 Amsterdam
- 我方需向卡车公司询价，加价后报给客户

**系统操作流程**：

```
1. Transport Requests → 新建
   ├── Request Type: Commercial
   ├── Cargo Type: Container
   ├── Destination: Terminal / Depot to Customer
   ├── Customer: ACME B.V.（阿姆斯特丹）
   ├── Origin Terminal: Rotterdam Maasvlakte Terminal
   ├── Delivery Address: Amsterdam, Strawinskylaan 100
   ├── 填写货物：1×40HQ, TEMU4567890, 20000kg
   └── 保存

2. 点击 "Start Inquiry" → 创建 Inquiry
   ├── 承运商: Van der Wal Transport B.V. → 回复 €850
   ├── 承运商: DVS Logistics → 回复 €790
   └── 选 DVS Logistics → Accept

3. 创建 Quote（从 accepted inquiry 自动预填成本）
   ├── Carrier Cost: €790（自动带入，表示承运商成本）
   ├── Margin Amount: €118.50（手工输入加价额）
   ├── Margin Rate: 15%（自动计算）
   ├── Total: €908.50（自动计算）
   ├── 发送给客户
   └── 客户接受 → 自动创建 Transport Order + 2条 fee.line
       ├── customer_charge €908.50 → 客户
       └── carrier_cost €790.00 → DVS Logistics
```

### 实例 2B：阿姆斯特丹货站 → 乌得勒支客户（托件）

**背景**：
- 空运到货 Amsterdam Schiphol Cargo Terminal

**系统操作流程**：

## 实例 5：我方仓库 A → 我方仓库 B（调拨）

### 实例 5A：鹿特丹 → 安特卫普（保税调拨）

**背景**：
- 我方 Rotterdam 保税仓有 20 托盘化工原料，调拨到 Antwerp 保税仓
- 涉及保税账册核销

**系统操作流程**：

```
1. Transport Requests → 新建
   ├── Request Type: Plan-Driven
   ├── Cargo Type: Pallet / Piece
   ├── Destination: Our Warehouse Transfer
   ├── Source Warehouse: Rotterdam
   ├── Destination Warehouse: Antwerp
   ├── 20 pallets, 16000 kg
   ├── Bonded Transfer: ✅（强制勾选）
   └── 保存

2. Go to Schedule → 拖到 7月14日
   └── 排期自动创建 pickup.plan

3. pickup.plan → Create Transport Order → 自动创建 fee.line (carrier_cost)
```

### 实例 5B：海牙 → 鹿特丹（非保税调拨）

```
1. Transport Requests → 新建（Plan-Driven, Warehouse Transfer）
   ├── 5 pallets, Source: Den Haag, Destination: Rotterdam
   └── Bonded Transfer: 不勾选

2. Schedule → 排期 → pickup.plan → Create Transport Order
```

### 实例 5C：鹿特丹 A 库 → 鹿特丹 B 库（同城短途）

## 实例 6：客户地址 → 我方仓库（退货回仓）

### 实例 6A：阿姆斯特丹客户 → 鹿特丹仓库退货

**背景**：
- 客户退回一批电子元件，2 托盘，800kg，由我司上门提货

**系统操作流程**：

```
1. Transport Requests → 新建
   ├── Request Type: Commercial
   ├── Cargo: Pallet, 2 pallets, 800 kg
   ├── Destination: Terminal / Depot to Customer
   ├── Destination Warehouse: Rotterdam（设置了 warehouse_id 则触发 reverse_logistics）
   ├── Customer: 阿姆斯特丹客户
   ├── Delivery Address: Amsterdam, pick up at customer site
   └── 保存

2. Start Inquiry → Quote → 客户确认 → Order + fee.line
   └── transport_type 自动设为 reverse_logistics
```

## 实例 8：Depot ↔ 仓库（空柜调拨）

### 实例 8A：ECT 堆场 → 鹿特丹仓库（提空柜）

**背景**：
- 我方需要 3 个 40HQ 空柜从 ECT Delta 堆场提到 Rotterdam 仓库

**系统操作流程**：

```
1. Container Service → 新建
   ├── Direction: Depot / Terminal → Warehouse
   ├── Container Record: MSCU1234567（从柜档案选择）
   ├── Quantity: 3
   ├── Depot: ECT Delta Terminal
   ├── Warehouse: Rotterdam
   ├── Carrier: Van der Wal Transport
   ├── Planned Date: 2026-07-11
   └── 保存

2. Confirm → Create Dispatch Order
   └── 自动生成 Transport Order
```

### 实例 8B：鹿特丹仓库 → RSC 堆场（还空柜）

```
1. Container Service → 新建
   ├── Direction: Warehouse → Depot / Terminal
   ├── Container: MAEU9876543 + MAEU5432198
   ├── Warehouse: Rotterdam
   ├── Depot: RSC Maasvlakte
   ├── Carrier: DVS Logistics
   └── 保存 → Confirm → Create Dispatch Order
```
