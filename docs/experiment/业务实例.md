## 实例 1：Terminal → 我方仓库（计划驱动）

### 实例 1A：鹿特丹码头 → 鹿特丹仓库（集装箱）

**背景**：
- 客户：ABC Trading B.V.
- 提单号：NKGA84065
- 到港码头：Rotterdam Maasvlakte Terminal
- 目的地：我方 Rotterdam 仓库
- 货物：2×40HQ 集装箱，电子产品

**系统操作流程**：

```
1. Pickup Requirements → 新建
   ├── Cargo Type: Container
   ├── Source: Manual
   ├── Destination: Terminal → Our Warehouse
   ├── Warehouse: Rotterdam
   ├── 填写 2 条集装箱行：
   │   ├── MSCU1234567 / 40HQ / 22000kg / BL: NKGA84065
   │   └── MSCU7890123 / 40HQ / 21500kg / BL: NKGA84065
   └── 保存

2. Schedule → 筛选 Warehouse → 拖拽到 7月10日
   ├── 两个柜同时排到 7/10
   └── scheduled_date 自动写入

3. Transportation Execution → Orders
   ├── 当前处于排期阶段
   └── (后续在排期页或订单页创建运输订单)
```

### 实例 1B：安特卫普码头 → 海牙仓库（托件）

**背景**：
- 客户：XYZ Logistics NV
- 到港：Antwerp Gateway Terminal

## 实例 2：Terminal → 客户地址（商务报价）

### 实例 2A：鹿特丹码头 → 阿姆斯特丹客户（集装箱）

**背景**：
- 客户向我方询价：从 Rotterdam 码头提一个 40HQ 柜送到 Amsterdam
- 我方需要向卡车公司询价，加价后报给客户

**系统操作流程**：

```
1. Pickup Requirements → 新建
   ├── Cargo Type: Container
   ├── Customer: ACME B.V.（阿姆斯特丹）
   ├── Destination: Terminal → Customer
   ├── Delivery: Amsterdam, Strawinskylaan 100
   ├── Container: 1×40HQ, TEMU4567890, 20000kg
   └── 保存

2. 点击 "Create Transport Request"
   └── 自动创建 Transport Request，预填货物信息

3. 在 Transport Request 中 → Inquiries → 新建
   ├── 向 Van der Wal Transport B.V. 询价 €850
   ├── 向 DVS Logistics 询价 €790
   └── 两家均回复

4. 选 DVS Logistics €790 → Quotes → 新建
   ├── 加 15% margin → €908.50
   ├── 发给客户
   └── 客户接受 → 自动创建 Transport Order
```

### 实例 2B：阿姆斯特丹货站 → 乌得勒支客户（托件）

**背景**：
- 空运到货 Amsterdam Schiphol Cargo Terminal
- 需派送到 Utrecht 客户仓库
- 3 托盘电子元件，800kg

**系统操作流程**：

```
1. Pickup Requirements → 新建
   ├── Cargo Type: Pallet / Piece
   ├── Customer: Utrecht Electronics B.V.
   ├── Destination: Terminal → Customer
   ├── 3 pallets, 800 kg
   └── 保存 → Create Transport Request

2. 承运商询价 → 我方报价 → 客户确认 → 运输订单
```

## 实例 3：我方仓库 → 客户地址（商务报价）

### 实例 3A：鹿特丹仓库 → 阿姆斯特丹客户（卡车配送）

**背景**：
- 我方 Rotterdam 仓库有 5 托盘食品原料
- 客户委托我们配送到 Amsterdam 的工厂
- 我方需找卡车公司承运

**系统操作流程**：

```
1. Pickup Requirements → 新建
   ├── Destination: Terminal → Customer

   ├── Customer: Amsterdam Food Factory
   ├── 5 pallets, 4000 kg, 12 m³
   └── 保存 → Create Transport Request

2. 询价 → 报价 → 订单（同实例 2）
```

## 实例 5：我方仓库 A → 我方仓库 B（调拨）

### 实例 5A：鹿特丹 → 安特卫普（保税调拨）

**背景**：
- 我方 Rotterdam 保税仓有 20 托盘化工原料
- 需要调拨到 Antwerp 保税仓
- 涉及保税账册核销

**系统操作流程**：

```
1. Pickup Requirements → 新建
   ├── Cargo Type: Pallet / Piece
   ├── Destination: Our Warehouse Transfer
   ├── Source Warehouse: Rotterdam
   ├── Warehouse: Antwerp
   ├── 20 pallets, 16000 kg
   ├── Bonded Transfer: ✅（强制勾选）
   └── 保存

2. Schedule → 拖到 7月14日
3. Create Transport Order
```

### 实例 5B：海牙 → 鹿特丹（非保税调拨）

**背景**：
- 海牙仓库有 5 托通用物资需调拨到鹿特丹仓库
- 非保税货物

```
1. 提货需求 → 仓库调拨
2. 不勾选 Bonded Transfer
3. 排期 → 生成运输订单
4. 安排卡车运输
```

### 实例 5C：鹿特丹 A 库 → 鹿特丹 B 库（同城调拨）

**背景**：
- 同城两个仓库之间的短途调拨
- 3 个 40HQ 空柜需要从 A 库移到 B 库

```
1. 柜类型，3 个 40HQ 空柜
2. Source: Rotterdam Warehouse A
3. Destination: Rotterdam Warehouse B
4. 排期 → 运输订单（短途，通常当日完成）
```

## 实例 6：客户地址 → 我方仓库（退货回仓）

### 实例 6A：阿姆斯特丹客户 → 鹿特丹仓库退货

- 客户退回一批电子元件，2 托盘，800kg

```
1. Pickup Requirements → 新建
   ├── Cargo Type: Pallet / Piece
   ├── Destination: Terminal → Customer（走商务报价）
   ├── 2 pallets, 800 kg
   ├── Delivery: Amsterdam, pick up at customer site
   └── 保存 → Create Transport Request
       └── transport_type = reverse_logistics
```

```
┌─────────────────────────────────────────────────┐
│ Container No.  │ Type │ Needs Swap │ Swap Location │
├─────────────────┼──────┼────────────┼───────────────┤
│ MSCU1234567     │ 40HQ │ ✅ yes     │ ECT Delta     │
└─────────────────┴──────┴────────────┴───────────────┘
```

## 实例 8：Depot ↔ 仓库（空柜调拨）

### 实例 8A：ECT 堆场 → 鹿特丹仓库（提空柜）

**背景**：
- 我方需要 3 个 40HQ 空柜从 ECT Delta 堆场提到 Rotterdam 仓库
- 用于下周装货出口

**系统操作流程**：

```
1. Container Service → 新建
   ├── Direction: Depot / Terminal → Warehouse
   ├── Container Record: MSCU1234567（从柜档案选择）
   ├── Quantity: 3
   ├── Depot: ECT Delta Terminal
   ├── Warehouse: Rotterdam
   ├── Carrier: Van der Wal Transport
   ├── Planned Date: 2026-07-11
   └── 保存

2. Confirm → Create Dispatch Order
   └── 自动生成 Transport Order
       └── transport_type = port_to_warehouse
       └── 容器明细写入 MSCU1234567
```

### 实例 8B：鹿特丹仓库 → RSC 堆场（还空柜）

**背景**：
- 卸货后有 2 个马士基的空柜需要还到 RSC Maasvlakte 堆场

```
1. Container Service → 新建
   ├── Direction: Warehouse → Depot / Terminal
   ├── Container: MAEU9876543 + MAEU5432198
   ├── Warehouse: Rotterdam
   ├── Depot: RSC Maasvlakte
   ├── Carrier: DVS Logistics
   └── 保存 → Confirm → Create Dispatch Order
       └── transport_type = to_customer（映射）
```

### 实例 8C：安特卫普仓库 → MSC 堆场（还重柜）

```
1. 空柜调拨申请
2. Direction: Warehouse → Depot
3. 1×40HC，柜号 MSCU7766554
4. 需换柜：Not needed（已提前完成）
5. 下单派车拖到 MSC Antwerp Depot
```
